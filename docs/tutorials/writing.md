# 写作规范

一份可持续的笔记体系，靠的是一致性。

## 标题结构

- 每页只保留一个 `#` 标题
- 内容段落用 `##` / `###`，便于右侧大纲提取

## “先可用，再优雅”

先把信息记录下来，再逐步把它打磨成教程：

1. 写在 `notes/` 的收件箱或当天记录里
2. 当某个主题出现 3 次以上，就抽到 `tutorials/` 形成结构化章节
3. 用“问题 → 结论 → 步骤 → 示例”的顺序重写

## 代码片段

```ts
type NoteMeta = {
  title: string
  tags?: string[]
}
```

## 提示块

::: tip 经验
把“能复用的步骤”写成教程，把“只影响当下的上下文”留在日常笔记里。
:::

::: warning 边界
当内容需要权限、团队协作或在线编辑时，可以在此基础上再引入后端与鉴权。
:::
