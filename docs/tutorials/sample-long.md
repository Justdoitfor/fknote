# 长文示例：从笔记到教程的最小闭环

这是一篇偏长的示例文章，用于观察：长段落可读性、章节间距、目录与大纲的协调、代码块密度、图片（可选）、以及分页/滚动体验。

## 背景

很多人的笔记系统“写得很勤”，但“复用很少”。核心原因通常不是工具，而是结构：内容缺少可阅读的路径，信息无法被快速定位，知识也就难以沉淀。

FKNote 的目标是用一个非常朴素的策略解决这个问题：把笔记写成可发布的文档，把文档组织成可连续阅读的教程。

## 最小闭环

一个可持续的闭环通常包含 4 步：

1. 收集：把碎片记录进收件箱
2. 归档：每周整理一次，把内容移动到主题或日期目录
3. 抽取：当某主题出现 3 次以上，把它抽成教程页
4. 固化：把教程挂到侧边栏，并补齐示例与常见问题

在写教程时，最常见的问题是“过度叙事”。写作不是复盘日志，而是交付方法。你可以保留必要的背景，但不要让背景吞掉结论；你可以保留必要的细节，但不要把细节变成流水账。最好的教程通常看起来很短，因为它只保留了能帮助读者完成任务的部分。

## 常见问题

### Q1：要不要一开始就写很完整？

不需要。先保证结构正确：标题层级清晰、侧边栏入口明确、正文可连续阅读。之后再逐步补齐细节。

### Q2：如何避免内容分散？

把“入口”当成产品功能：侧边栏不要太深，优先保证从首页 2 次点击内到达常用内容。

### Q3：如何做标签？

如果你不需要全站聚合检索，路径分层往往比标签更稳、更可维护。
