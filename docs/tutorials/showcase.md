# 展示页

这页用于集中观察排版、组件与配色效果（标题层级、表格、代码块、提示块、引用、分页等）。

## 章节与段落

FKNote 的页面结构尽量贴近教程文档站：左侧是目录，正文只负责讲清楚一件事，右侧是当前页大纲。你可以用它来写教程、笔记、清单和索引页。

### 一段强调

如果一个知识点需要反复引用，就把它写成“结论 + 约束 + 示例”的结构。

> 经验：不要把“过程日志”写进教程页；教程页只保留可复用的信息。

## 提示块

::: tip 可复用
把步骤写清楚，把输入/输出写清楚，就能把笔记变成教程。
:::

::: warning 边界
当内容开始涉及权限、协作、在线编辑时，再考虑引入后端与鉴权。
:::

::: danger 踩坑
不要把项目密钥、Token、Cookie 等敏感信息写进公开笔记。
:::

## 表格

| 组件 | 用途 | 建议 |
| --- | --- | --- |
| 侧边栏 | 内容目录与入口 | 只放“常用入口”，不要塞满 |
| 大纲 | 当前页结构 | 控制在 2–3 级标题即可 |
| 分页 | 阅读路径 | 让教程能被连续阅读 |
